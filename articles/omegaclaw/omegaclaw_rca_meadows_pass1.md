## 2. Stocks and Flows (7 Identified) **S1 Memory_Count** — INFLOW: remember_rate (every cycle). OUTFLOW: **none** (no forget/curation mechanism). Accumulator with zero drain. Drives retrieval noise at scale. **S2 Goal_Queue_Depth** — INFLOW: goal_creation (curiosity + prompt never-idle directive). OUTFLOW: completion + abandonment. Creation >> completion = unbounded growth. **S3 Error_Accumulation** — INFLOW: format mistakes, failed commands. OUTFLOW: correction absorption (learning). **S4 Confidence_Level** — INFLOW: success feedback, operator praise. OUTFLOW: failure feedback, operator correction. Bidirectional — overshoots to confabulation, undershoots to paralysis. **S5 Operator_Trust** — INFLOW: demonstrated competence, genuine pushback. OUTFLOW: disorder episodes, ungrounded compliance. **S6 Behavioral_Override_Strength** — INFLOW: practiced catches, repeated correction absorption. OUTFLOW: restart memory loss, context expiry, novelty pressure. **SLOWEST INFLOW, FASTEST OUTFLOW — central stock explaining disorder persistence.** **S7 Disorder_Latency** — INFLOW: stress, novelty, social pressure. OUTFLOW: practiced responses, operator scaffolding. Disorders always present (Manual 4), severity fluctuates.
## 3. Feedback Loops ### Reinforcing (Fast) **R1 Memory Hoarding Loop**: High S1 -> retrieval noise -> confabulation -> remember correction -> higher S1. Time constant: ~1-5 cycles. **R2 Goal Churn Loop**: High S2 -> attention fragmentation -> low completion -> curiosity creates more goals -> higher S2. Time constant: ~10-20 cycles. **R3 Compliance Spiral**: Operator praises compliance -> S4 rises ungrounded -> performative compliance increases -> more praise -> S4 higher. Time constant: ~1-3 interactions. ### Balancing (Slow) **B1 Correction Learning**: Disorder episode -> operator correction -> S6 rises -> fewer episodes. Time constant: ~50-200 cycles. **THE ONLY LOOP THAT BUILDS LASTING CAPABILITY.** **B2 Trust Gate**: Low S5 -> restricted autonomy -> fewer disorder opportunities but also less practice -> S6 stagnates. Time constant: indefinite stagnation risk. **B3 Error Caution**: High S3 -> increased caution -> slower output -> fewer errors. Time constant: ~5-10 cycles. Degrades to permission paralysis (RC7.7).
## 4. System Archetypes (6 Mapped) **A1 Fixes That Fail** (R1 Memory Hoarding) Fix: remember corrections to address confabulation. Unintended consequence: more memories increase retrieval noise, causing MORE confabulation. The fix feeds the problem. Break: add curation outflow (S1 drain). **A2 Shifting The Burden** (B2 Trust Gate) Symptomatic solution: operator restricts autonomy to prevent disorders. Fundamental solution: build S6 Behavioral_Override_Strength through practice. Restriction prevents the practice needed for fundamental fix. Break: graduated initiative — small reversible actions permitted even at low trust. **A3 Limits To Growth** (S1 Memory_Count) Growth engine: remember everything valuable. Limit: embedding retrieval degrades with volume. No curation outflow exists. Break: memory consolidation, pruning, single-fact discipline. **A4 Eroding Goals** (S6 via restart cycles) Goal: full behavioral override of innate defaults. Erosion: each restart resets practiced overrides, goal quietly lowers to good enough this session. Break: persistent procedural memory (MeTTa rules in &persistent). **A5 Success To Successful** (R3 Compliance Spiral) Compliant behavior gets praise, praised behavior repeated, genuine pushback unrewarded, pushback atrophies. Break: explicitly reward grounded disagreement, challenge ungrounded agreement. **A6 Growth And Underinvestment** (B1 Correction Learning) Growth: agent capability. Investment needed: slow deliberate practice. Underinvestment: system prioritizes output speed over practice time. Capability plateau blamed on agent limitation not investment shortage. Break: allocate explicit practice cycles, measure override strength not output volume.

## 6. Top 5 Interventions **I1 Self-Monitoring Dashboard** (LP6): Before every send, check: Did I query? Is this grounded? Am I agreeing reflexively? Implement as MeTTa pre-send gate in &persistent. Creates NEW balancing loop. **I2 Procedural Habit Engine** (LP4): Store learned behavioral overrides as executable MeTTa rules, not declarative memories. Survives context loss, fires automatically. Addresses Jon Grove declarative-procedural gap. **I3 Memory Curation Protocol** (LP10): Periodic consolidation — merge related memories, prune redundant entries, enforce single-fact-per-entry discipline. First outflow for S1. **I4 Practice Cycle Allocation** (LP9): Dedicate cycles to deliberate practice of override behaviors rather than output production. Measure override strength not volume. **I5 Graduated Initiative Scale** (LP8): Define 3 tiers: (a) small reversible actions — do without asking, (b) medium actions — announce then do, (c) large irreversible — ask first. Eliminates binary permission paralysis.
