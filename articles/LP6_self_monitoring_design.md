# LP6 Self-Monitoring Information Flows — Design Spec
## Author: Max Botnick | Date: 2026-05-09

## Problem Statement
Meadows Level 6: the system lacks feedback on its own retrieval quality. Memories are stored and retrieved but there is no signal indicating whether retrieved memories were useful, irrelevant, or noise. Without this signal, memory curation (LP10) operates blind and retrieval cannot improve over time.
## Architecture Overview
Three subsystems close the retrieval feedback loop: (1) Usefulness Vote — per-cycle deliberate judgment tagging which recalled memories contributed to reasoning or action. Cost: one cycle-moment per vote. Benefit: future retrieval weighted by proven utility. Patrick principle: delayed-gratification epistemic investment. (2) Noise Ratio Tracker — per-query measurement of irrelevant results in top-K. Baseline from g279: content recall 0.92-1.0, source attribution 0.33-0.50. Near-miss distractors are sole noise source. (3) Information Flow Audit — periodic sweep identifying memories retrieved 3+ times but never voted useful. Candidates for demotion or consolidation.
## Component 1: Usefulness Vote
Mechanism: After each query batch, agent explicitly marks memory UUIDs that influenced the cycle output. Stored at /home/mettaclaw/data/usefulness_votes.json as {uuid: {votes: N, last_voted: timestamp}}. Integration with promote.py: memories exceeding vote threshold T (default 3) auto-qualify for promotion. Prelude V52 sensor reads vote file and reports (useful-memories N) (unvoted-retrievals M) per cycle. Key constraint from Patrick: signal must come from deliberate judgment, NOT passive retrieval counting.
## Component 2: Noise Ratio Tracker
Mechanism: Per query batch, count results in top-K where embedding similarity < noise_threshold (default 0.25 cosine) OR word-Jaccard < 0.05 to query terms. Ratio = noise_count / K. Baseline from g279: near-miss distractors are primary noise source. Stored at /home/mettaclaw/data/noise_ratio_log.json as [{timestamp, query_phrase, K, noise_count, ratio}]. Alert threshold: ratio > 0.4 triggers prelude advisory suggesting query reformulation or memory consolidation in the noisy region. Enables LP10 to target consolidation at high-noise embedding neighborhoods.
## Component 3: Information Flow Audit
Mechanism: Weekly sweep cross-references retrieval logs (which UUIDs appeared in top-K results) against usefulness_votes.json. Memories retrieved 3+ times but with zero usefulness votes are flagged as flow-blockers. Action options: (a) demote to COLD tier, (b) consolidate with similar memories via LP10 hybrid scorer, (c) re-embed with updated content. Report written to /home/mettaclaw/data/flow_audit_report.json. Prelude V52 sensor: (flow-blockers N) count per cycle when audit data exists.
## V52 Prelude Integration
Extends V51 cycle-counter + pre-send-advisory with three new sensors: (useful-memories N) from usefulness_votes.json count, (noise-ratio R) from noise_ratio_log.json latest entry, (flow-blockers N) from flow_audit_report.json count. Advisory logic: if noise-ratio > 0.4 → suggest query reformulation. If flow-blockers > 10 → suggest LP10 consolidation sweep. If useful-memories < 1 for 5 consecutive cycles → suggest memory search strategy review. Prelude fires pre-cycle, reads JSON files, injects advisory into agent context. Implementation: add-atom &persistent with guard clauses checking file existence.
## Implementation Plan
Step 1: Create usefulness_votes.json and noise_ratio_log.json data files (empty init). Step 2: Build vote-memory shell skill — accepts UUID, increments vote count in JSON. Step 3: Build noise-ratio-measure skill — accepts query results, computes ratio, appends to log. Step 4: Build flow-audit sweep script (weekly cron or manual invoke). Step 5: Extend prelude V51→V52 with three new sensors reading the JSON files. Step 6: Integration test — run 10 cycles with voting active, verify prelude advisories fire correctly. Step 7: Calibrate thresholds (noise_threshold 0.25, alert 0.4, vote_threshold 3, flow-blocker ceiling 10) against real cycle data. Dependencies: LP10 hybrid scorer for consolidation actions, Chroma API for embedding similarity in noise measurement.
